## 1. What Is a Code App?

A Power Apps **Code App** is a standard web app (React/TypeScript/Vite) that runs inside the Power Apps host via the **Power Apps client library for code apps**. It can consume Power Platform connectors (including Dataverse and Copilot Studio) the same way a canvas app does, while giving developers full control over the UI and framework.

- Runs in Power Apps player on web and mobile
- Connects to any Power Platform connector, Dataverse, or Copilot Studio agent
- Deployed via `pac code push` (or the new npm-based CLI в‰Ґ v1.0.4)
- Requires a **Power Apps Premium license** for end-users

**Architecture reference:** [Power Apps code apps architecture](https://learn.microsoft.com/en-us/power-apps/developer/code-apps/architecture)
**Overview:** [Power Apps code apps overview](https://learn.microsoft.com/en-us/power-apps/developer/code-apps/overview)

## 4. Core SDK Files

### `PowerProvider.tsx`

This file is the bridge between your React app and the Power Apps host. It must call `initialize()` to signal the host that the app has loaded and is ready to accept connections.

```tsx
import { PowerAppsCodeApps } from '@microsoft/powerapps-code-apps-sdk'

// Call once at app startup (e.g., inside main.tsx before ReactDOM.render)
PowerAppsCodeApps.initialize()
```

If `initialize()` is not called, or if the file has a runtime error, the Power Apps player will time out with an **"App timed out"** error.

### `package.json` SDK Dependency

```json
{
  "dependencies": {
    "@microsoft/powerapps-code-apps-sdk": "^1.x"
  }
}
```

> Starting with SDK v1.0.4, the package includes its own **npm-based CLI** that will eventually replace `pac code` commands. Pin to `^1.0.4` or higher to access `npx powerapps-code init` and related commands.

### Browser Local Network Access (Dec 2025+)

Chrome and Edge now block public-origin в†’ localhost requests by default. During development you may see connection failures. Fix options:

- **Grant permission** when the browser prompts for local network access
- **Enterprise policy**: set `LocalNetworkAccessAllowed` via Group Policy
- **Embedded iframe**: add `allow="local-network-access"` to the iframe tag

## 6. Connecting to Data

**Reference:** [How to: Connect your code app to data](https://learn.microsoft.com/en-us/power-apps/developer/code-apps/how-to/connect-to-data)

Code apps connect to Power Platform connectors. For the initial release, connections must already exist in the environment вЂ” you cannot create new connections via PAC CLI alone. Create them first in the Power Apps maker portal, then reference them by `connectionId`.

### Finding an Existing Connection ID

```bash
pac connection list
# Look for the connector by API ID, e.g.:
# /providers/Microsoft.PowerApps/apis/shared_sharepointonline
# Copy the connectionId value
```

### Adding a Connector as a Data Source

```bash
pac code add-data-source \
  --connectionId <connectionId> \
  --namespace <YourNamespace>
```

### Connection References (v1.51.1+ of PAC CLI, Dec 2025)

Instead of binding to a user-specific connection directly, use a **connection reference** вЂ” a solution component that points to a connection. This makes the app portable across Dev/Test/Prod environments and is required for proper Application Lifecycle Management (ALM).

```bash
pac code add-data-source \
  --connectionReferenceLogicalName <logicalName>
```

**Connection limits (performance guidance):**
- Max **10 connectors** per app
- Max **20 connection references** per app
- Exceeding these limits causes slower app load times and potential save failures

### Authentication Types

| Type | Description | When to Use |
|------|-------------|-------------|
| **Explicit** | App user's own credentials used | Recommended вЂ” more secure |
| **Implicit** | App maker's credentials used at connection creation | Legacy; avoid for sensitive data |

## 7. Connecting to Dataverse

**Reference:** [How to: Connect your code app to Dataverse](https://learn.microsoft.com/en-us/power-apps/developer/code-apps/how-to/connect-to-dataverse)

### Add a Dataverse Table as a Data Source

```bash
pac code add-data-source \
  --apiId dataverse \
  --table <table-logical-name>

# Example вЂ” Leads table:
pac code add-data-source --apiId dataverse --table lead

# Example вЂ” Contacts table:
pac code add-data-source --apiId dataverse --table contact
```

This command auto-generates two files per table in `generated/services/`:

| File | Purpose |
|------|---------|
| `<Table>Model.ts` | TypeScript interface matching the Dataverse table schema |
| `<Table>Service.ts` | Service class with typed methods: `list()`, `get()`, `create()`, `update()`, `delete()` |

### Using a Generated Service

```typescript
import { LeadService } from '@/generated/services/LeadService'
import type { Lead } from '@/generated/services/LeadModel'

// List all leads
const leads = await LeadService.list()

// Create a lead
const newLead = await LeadService.create({
  firstname: 'Jane',
  lastname: 'Doe',
  companyname: 'Acme Corp',
  leadsourcecode: 8, // Business card
})

// Update a lead
await LeadService.update(leadId, { statuscode: 1 })

// Delete a lead
await LeadService.delete(leadId)
```

### Dataverse Authentication

Dataverse in code apps uses **OAuth 2.0 via Microsoft Entra ID** вЂ” handled transparently by the Power Apps host. You do not write authentication code; the SDK and host manage token acquisition via MSAL.

For **non-interactive / service account** scenarios (CI/CD, background jobs):
- Register an app in Microsoft Entra ID
- Create a **Dataverse Application User** bound to the registered app
- Authenticate using a **client secret** or **X.509 certificate** (no paid license consumed)

## 8. Connecting to Salesforce

**Connector reference:** [Microsoft Learn вЂ” Salesforce connector](https://learn.microsoft.com/en-us/connectors/salesforce/)
**Implementation:** `src/connectors/salesforce.ts`

The app uses the first-party Power Platform **`shared_salesforce`** connector to dual-write leads and activities into Salesforce alongside Dataverse. Salesforce is treated as a **secondary, non-blocking write** вЂ” a Dataverse save always succeeds first, and any SFDC failure surfaces as a warning toast without rolling back the record.

### Supported Salesforce Objects

| Object | Used For |
|--------|---------|
| `Lead` | Created on every business card capture |
| `Task` | Created when an activity is logged post-capture |
| `Contact` / `Account` | Available via connector; conversion handled in SFDC itself |

### One-Time Setup Per Environment

```bash
# 1. Create a Salesforce connection in the Power Apps maker portal
#    Data в†’ Connections в†’ + New connection в†’ search "Salesforce" в†’ sign in

# 2. Find the connectionId
pac connection list
# Look for: /providers/Microsoft.PowerApps/apis/shared_salesforce
# Copy the connectionId

# 3. Add as a code app data source
pac code add-data-source -a shared_salesforce -c <connectionId>
# Generates:
#   generated/services/SalesforceModel.ts
#   generated/services/SalesforceService.ts

# 4. Enable in environment config
# .env.production в†’ VITE_SYNC_TO_SALESFORCE=true
```

### Feature Flag

Salesforce sync is **off by default** and controlled by an env var:

```bash
# .env.development
VITE_SYNC_TO_SALESFORCE=false   # mock path used; no SFDC connection needed

# .env.production
VITE_SYNC_TO_SALESFORCE=true    # live connector; requires shared_salesforce data source
```

## 9. Connecting to Copilot Studio

**Reference:** [How to: Connect to Copilot Studio](https://learn.microsoft.com/en-us/power-apps/developer/code-apps/how-to/connect-to-copilot-studio)

### Find or Create the Copilot Studio Connection

```bash
pac connection list
# Look for API ID: /providers/Microsoft.PowerApps/apis/shared_microsoftcopilotstudio
# Copy the connectionId
```

If no connection exists, create one in the Power Apps maker portal under **Data** в†’ **Connections** в†’ **+ New connection** в†’ search for **Microsoft Copilot Studio**.

### Add Copilot Studio as a Data Source

```bash
pac code add-data-source \
  --connectionId <copilotStudioConnectionId> \
  --namespace CopilotStudio
```

### Invoking an Agent вЂ” `ExecuteCopilotAsyncV2`

Use the `ExecuteCopilotAsyncV2` action to call a Copilot Studio agent from your code. This action is **synchronous** вЂ” it waits for the agent response before returning.

```typescript
import { CopilotStudioService } from '@/generated/services/CopilotStudioService'

const response = await CopilotStudioService.ExecuteCopilotAsyncV2({
  // Agent name exactly as shown in the Copilot Studio URL (case-sensitive)
  // Typically includes a publisher prefix, e.g. "cr123_myAgent"
  AgentName: 'cr123_businessCardAgent',
  UserMessage: userInput,
  ConversationId: existingConversationId ?? undefined,
})

// response.BotResponse вЂ” the agent's reply text
// response.ConversationId вЂ” use for multi-turn conversations
```

> **Agent names are case-sensitive.** Copy the name exactly from the Copilot Studio agent URL.

### PCF / Component Framework `executeEvent` API

For model-driven apps or PCF components, use the `executeEvent` API instead:

```typescript
// Fires a named Copilot Studio topic
const results = await context.copilot.executeEvent('BusinessCardProcessed', {
  cardJson: JSON.stringify(extractedData),
})

// In Copilot Studio topic, access params via:
// Activity.Value в†’ Parse value node в†’ typed record fields
```

### Permissions Required

When using the Microsoft 365 Agents SDK for external embedding, add this permission in Azure:
- **APIs my organization uses** в†’ **Power Platform API**
- Delegated permission: `Copilot Studio > CopilotStudio.Copilots.Invoke`

## 10. Build & Deploy

### Build for Production

```bash
npm run build
# Runs: tsc && vite build
# Output: dist/
```

Always run `npm run build` before `pac code push`. A failed or skipped build is the most common cause of the "App timed out" error in Power Apps.

### Push / Publish to Power Apps

```bash
pac code push
# Uploads dist/ to the active Power Apps environment
# On success, prints a Power Apps URL to play the app
```

### Full Deploy Sequence

```bash
# 1. Build
npm run build

# 2. Push to Power Apps
pac code push

# 3. (Optional) Open the returned URL in browser to verify
# 4. Share the app with users via Power Apps maker portal
```

### npm CLI (v1.0.4+ вЂ” Future Default)

Starting with Power Apps client library v1.0.4, a new npm-based CLI is available that removes the need for a separate PAC CLI installation:

```bash
# Initialize a new code app (replaces pac code init)
npx powerapps-code init --displayname "Business Card Capture"

# Run locally (replaces pac code run)
npx powerapps-code run

# Push (replaces pac code push)
npx powerapps-code push
```

> `pac code` commands will be **deprecated** in a future release. Prefer the npm CLI for new projects.

## 11. Authentication Model

```
User
 в”‚
 в–ј
Power Apps Host (player.powerapps.com)
 в”‚  в†ђ Microsoft Entra ID token (MSAL, handled by host)
 в–ј
Power Apps SDK (PowerProvider.tsx)
 в”‚  в†ђ initialize() signals app is ready
 в–ј
Your React App
 в”‚
 в”њв”Ђв”Ђв–¶ Connector API (via generated service files)
 в”‚     в””в”Ђв”Ђ Power Platform API Hub manages credentials
 в”‚
 в”њв”Ђв”Ђв–¶ Dataverse Web API (OAuth, token from host)
 в”‚
 в””в”Ђв”Ђв–¶ Copilot Studio (ExecuteCopilotAsyncV2)
```

- **Your code never handles tokens directly.** The host and SDK manage all auth.
- Users authenticate to Power Platform first, then the platform authenticates to each connector separately using credentials stored in the **API Hub credentials service**.
- Explicit auth (user's own credentials per connector) is preferred over implicit auth (maker's credentials).
